## Problem Statement

Given an array of strings `strs`, group all **anagrams** together into sublists. You may return the output in **any order**.

An **anagram** is a string that contains the exact same characters as another string, but the order of the characters may differ.

<div className="callout">
  <p><strong>Example 1:</strong><br/>
  Input: <code>strs = ["eat","tea","tan","ate","nat","bat"]</code><br/>
  Output: <code>[["bat"],["nat","tan"],["ate","eat","tea"]]</code></p>
</div>

<div className="callout">
  <p><strong>Example 2:</strong><br/>
  Input: <code>strs = [""]</code><br/>
  Output: <code>[[""]]</code></p>
</div>

<div className="callout warning">
  <p><strong>Constraints:</strong> 1 ≤ strs.length ≤ 10⁴ · 0 ≤ strs[i].length ≤ 100 · strs[i] consists of lowercase English letters.</p>
</div>

## Intuition — Building the Insight

<Spoiler title="Reveal Core Intuition">

The challenge is finding a **canonical form** — a transformation where all anagrams produce the same output. Two strings are anagrams if and only if they contain the same characters at the same frequencies.

<div className="insight">
  <div className="insight-label">Core Insight</div>
  <p>Two approaches to canonicalize: (1) <strong>sort the string</strong> — all anagrams sort identically. (2) <strong>count character frequencies</strong> — all anagrams produce the same 26-element count array. Use either as a hash map key to group anagrams together.</p>
</div>

Sorting is simpler to implement (O(k log k) per string). Frequency counting is faster (O(k) per string) and is the optimal solution.

</Spoiler>

### Approach 1 — Brute Force

For each string, compare it against every other string using a helper that checks if they're anagrams. Group matching strings together. Messy and slow.

```python
def groupAnagrams(strs):
    def is_anagram(s1, s2):
        return sorted(s1) == sorted(s2)

    visited = [False] * len(strs)
    result = []

    for i in range(len(strs)):
        if visited[i]:
            continue
        group = [strs[i]]
        visited[i] = True
        for j in range(i + 1, len(strs)):
            if not visited[j] and is_anagram(strs[i], strs[j]):
                group.append(strs[j])
                visited[j] = True
        result.append(group)
    return result
```

**Complexity:** Time O(m² · k log k) · Space O(m · k). Too slow for 10⁴ strings.

### Approach 2 — HashMap with Sorted Key

Sort each string's characters to produce its canonical key. All anagrams produce the same sorted key, so they land in the same bucket.

```python
from collections import defaultdict

def groupAnagrams(strs):
    groups = defaultdict(list)

    for s in strs:
        key = "".join(sorted(s))   # "eat" → "aet", "tea" → "aet"
        groups[key].append(s)

    return list(groups.values())
```

**Complexity:** Time O(m · k log k) · Space O(m · k). Clean and interview-friendly.

### Approach 3 — HashMap with Frequency Key ★ Optimal

Instead of sorting, build a 26-element character count array for each string. Convert it to a tuple (tuples are hashable, lists are not) and use it as the key.

<div className="insight">
  <div className="insight-label">Why tuple, not list?</div>
  <p>Python dict keys must be hashable. Lists are mutable so they're not hashable. Tuples are immutable — <code>tuple(count)</code> converts your count array into a valid dict key in O(k) time.</p>
</div>

```python
from collections import defaultdict

def groupAnagrams(strs):
    groups = defaultdict(list)

    for s in strs:
        count = [0] * 26
        for c in s:
            count[ord(c) - ord('a')] += 1

        key = tuple(count)   # e.g. "eat" → (1,0,0,0,1,0,...,1,0,...) for a,e,t
        groups[key].append(s)

    return list(groups.values())
```

**Walkthrough for `strs = ["eat","tea","tan"]`:**
- "eat" → count for a=1, e=1, t=1 → key=(1,0,0,0,1,...,1,...) → group A
- "tea" → same counts → same key → group A
- "tan" → count for a=1, n=1, t=1 → different key → group B

**Complexity:** Time O(m · k) · Space O(m · k). Optimal.

## Edge Cases to Test

<div className="callout">
  <p><strong>1. Empty string:</strong> <code>[""]</code> → <code>[[""]]</code>. An empty string has all-zero counts and sorts to <code>""</code>. It forms its own group.</p>
</div>
<div className="callout">
  <p><strong>2. Single characters:</strong> <code>["a","b","a"]</code> → <code>[["a","a"],["b"]]</code>. Two "a"s are anagrams of each other.</p>
</div>
<div className="callout warning">
  <p><strong>3. Same string repeated:</strong> <code>["eat","eat"]</code> → <code>[["eat","eat"]]</code>. Identical strings are trivially anagrams — they land in the same group.</p>
</div>
