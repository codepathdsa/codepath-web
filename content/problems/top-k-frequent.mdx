## Problem Statement

Given an integer array `nums` and an integer `k`, return the `k` **most frequent elements**. You may return the answer in **any order**.

<div className="callout">
  <p><strong>Example 1:</strong><br/>
  Input: <code>nums = [1,1,1,2,2,3]</code>, <code>k = 2</code><br/>
  Output: <code>[1,2]</code></p>
</div>

<div className="callout">
  <p><strong>Example 2:</strong><br/>
  Input: <code>nums = [1]</code>, <code>k = 1</code><br/>
  Output: <code>[1]</code></p>
</div>

<div className="callout warning">
  <p><strong>Constraints:</strong> 1 ≤ nums.length ≤ 10⁵ · -10⁴ ≤ nums[i] ≤ 10⁴ · k is in range [1, number of unique elements] · The answer is guaranteed to be unique.</p>
</div>

## Intuition — Building the Insight

<Spoiler title="Reveal Core Intuition">

The brute-force path: count frequencies, sort by frequency descending, return first k. That's O(n log n). But can we do better?

<div className="insight">
  <div className="insight-label">Core Insight</div>
  <p>Frequencies range from 1 to n. Create an array of n+1 buckets where <code>bucket[freq]</code> holds all numbers that appear exactly <code>freq</code> times. Walk the buckets from high to low, collecting elements until you have k. This is <strong>Bucket Sort</strong> — O(n) time, no comparison sort needed.</p>
</div>

The key observation: you don't need to sort all elements. You just need the top k by frequency, and frequencies are bounded by n.

</Spoiler>

### Approach 1 — HashMap + Sort

Count frequencies with a hash map, then sort the unique elements by frequency descending, and return the first k.

```python
from collections import Counter

def topKFrequent(nums, k):
    count = Counter(nums)   # {1: 3, 2: 2, 3: 1} for [1,1,1,2,2,3]

    # Sort unique elements by frequency descending, take first k
    return sorted(count.keys(), key=lambda x: -count[x])[:k]
```

**Complexity:** Time O(n log n) · Space O(n). Simple but the sort adds unnecessary cost.

### Approach 2 — HashMap + Min-Heap

Maintain a min-heap of size k. For each element's frequency, if the heap has fewer than k items, push. If the new frequency is greater than the heap's minimum, replace it.

```python
import heapq
from collections import Counter

def topKFrequent(nums, k):
    count = Counter(nums)
    # heapq.nlargest uses a heap internally — O(n log k)
    return heapq.nlargest(k, count.keys(), key=count.get)
```

**Complexity:** Time O(n log k) · Space O(n). Better than sorting when k is small.

### Approach 3 — Bucket Sort ★ Optimal

Frequencies range from 1 to n. Create `n+1` buckets. Place each number into `bucket[frequency]`. Walk buckets from right to left (highest frequency first) to collect the top k elements.

<div className="insight">
  <div className="insight-label">Why O(n)?</div>
  <p>Building buckets takes O(n). Scanning them takes O(n). No comparison-based sort is involved — we're exploiting the known range of frequency values to avoid sorting entirely.</p>
</div>

```python
from collections import Counter

def topKFrequent(nums, k):
    count = Counter(nums)

    # bucket[i] = list of numbers that appear exactly i times
    bucket = [[] for _ in range(len(nums) + 1)]
    for num, freq in count.items():
        bucket[freq].append(num)

    result = []
    # Walk from highest frequency to lowest
    for freq in range(len(bucket) - 1, 0, -1):
        for num in bucket[freq]:
            result.append(num)
            if len(result) == k:
                return result

    return result
```

**Walkthrough for `nums = [1,1,1,2,2,3]`, `k = 2`:**
- Counts: `{1:3, 2:2, 3:1}`
- Buckets: `[[], [3], [2], [1], [], [], []]`
- Scan right-to-left: freq 3 → `[1]`, freq 2 → `[1,2]` → k=2 reached → return `[1,2]` ✓

**Complexity:** Time O(n) · Space O(n). Optimal.
