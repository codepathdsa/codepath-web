## Problem Statement

Given an integer array `nums`, return `true` if any value appears **at least twice** in the array, and return `false` if every element is distinct.

  <p><strong>Example 1:</strong><br/>
  Input: <code>nums = [1,2,3,1]</code><br/>
  Output: <code>true</code></p>
</div>

<div className="callout">
  <p><strong>Example 2:</strong><br/>
  Input: <code>nums = [1,2,3,4]</code><br/>
  Output: <code>false</code></p>
</div>

<div className="callout warning">

## Intuition — Building the Insight

<Spoiler title="Reveal Core Intuition">

The naive approach asks: "have I seen this number before?" If you scan the whole array from scratch for each element, that's O(n²). The insight is to **remember what you've seen** as you go.

<div className="insight">
  <div className="insight-label">Core Insight</div>
  <p>A hash set stores elements in O(1). As you walk the array, check if the current element is already in your set. If yes — duplicate found. If no — add it and continue. One pass, done.</p>
</div>

This is almost identical to Two Sum's complement-lookup trick: instead of looking up `target - x`, you're looking up `x` itself.

</Spoiler>

### Approach 1 — Brute Force

For each element, check every other element after it. Return `true` if any match is found.

```python
def containsDuplicate(nums):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] == nums[j]:
                return True
    return False
```

**Complexity:** Time O(n²) · Space O(1). Correct but impractical for n up to 10⁵.

### Approach 2 — Sort + Linear Scan

Sort the array so that duplicate values sit next to each other, then do a single pass checking adjacent elements.

```python
def containsDuplicate(nums):
    nums.sort()
    for i in range(len(nums) - 1):
        if nums[i] == nums[i + 1]:
            return True
    return False
```

**Complexity:** Time O(n log n) · Space O(1). Better, but modifies the original array.

### Approach 3 — Hash Set (One Pass) ★ Optimal

Iterate once. For each number, check if it's already in a set. If yes, it's a duplicate. If no, add it.

<div className="insight">
  <div className="insight-label">One-liner variant</div>
  <p>Because a set only stores unique values, <code>len(set(nums)) != len(nums)</code> is true if and only if duplicates were removed. This is the same logic compressed into a single expression.</p>
</div>

```python
def containsDuplicate(nums):
    seen = set()
    for num in nums:
        if num in seen:
            return True
        seen.add(num)
    return False
```

```python
# One-liner (same O(n) time, builds full set before checking)
def containsDuplicate(nums):
    return len(set(nums)) != len(nums)
```

**Complexity:** Time O(n) · Space O(n). Optimal.
