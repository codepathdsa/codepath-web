## Problem Statement

Given an unsorted array of integers `nums`, return the length of the **longest consecutive elements sequence**.

You must write an algorithm that runs in **O(n) time**.

<div className="callout">
  <p><strong>Example 1:</strong><br/>
  Input: <code>nums = [100,4,200,1,3,2]</code><br/>
  Output: <code>4</code><br/>
  Explanation: The longest consecutive sequence is <code>[1,2,3,4]</code>.</p>
</div>

<div className="callout">
  <p><strong>Example 2:</strong><br/>
  Input: <code>nums = [0,3,7,2,5,8,4,6,0,1]</code><br/>
  Output: <code>9</code></p>
</div>

<div className="callout warning">

</div>

## Intuition — Building the Insight

<Spoiler title="Reveal Core Intuition">

The O(n log n) path is obvious: sort, then scan for consecutive runs. The O(n) insight is less obvious.

<div className="insight">
  <div className="insight-label">Core Insight</div>
  <p>Only start counting a sequence at its <strong>true beginning</strong> — a number <code>n</code> where <code>n-1</code> does NOT exist in the set. From there, count up: n+1, n+2, ... until the chain breaks. This ensures every element is visited at most twice total: once in the outer loop, once in the inner count. That's O(n).</p>
</div>

Without this "start only at sequence heads" trick, you'd wastefully restart counting from numbers that are in the middle of a sequence.

</Spoiler>

### Approach 1 — Brute Force

For each number, try to build the longest sequence starting from it by checking if `n+1`, `n+2`, ... exist using a linear scan.

```python
def longestConsecutive(nums):
    longest = 0
    num_set = set(nums)

    for n in nums:
        length = 1
        while (n + length) in num_set:  # linear scan per start → O(n²) total
            length += 1
        longest = max(longest, length)

    return longest
```

**Complexity:** Time O(n²) · Space O(n). Using a set makes lookups O(1), but we still restart from every number.

### Approach 2 — Sort + Scan

Sort to bring consecutive numbers adjacent. Then scan once, counting current streak and resetting on gaps.

```python
def longestConsecutive(nums):
    if not nums:
        return 0

    nums.sort()
    longest = 1
    current = 1

    for i in range(1, len(nums)):
        if nums[i] == nums[i - 1]:
            continue          # skip duplicates
        elif nums[i] == nums[i - 1] + 1:
            current += 1
            longest = max(longest, current)
        else:
            current = 1       # gap — reset streak

    return longest
```

**Complexity:** Time O(n log n) · Space O(1). Clean, but the sort disqualifies it for the O(n) requirement.

<Premium>

### Approach 3 — Hash Set + Sequence Heads ★ Optimal

Load all numbers into a set for O(1) lookups. For each number, check if it's a **sequence head** (i.e., `n-1` is not in the set). Only from heads do we count upward.

<div className="insight">
  <div className="insight-label">Why it's truly O(n)</div>
  <p>Every number either: (a) gets skipped immediately as a non-head, or (b) is counted exactly once as part of exactly one sequence. Across the entire input, the inner <code>while</code> loop runs a total of n iterations — not n per outer loop.</p>
</div>

```python
def longestConsecutive(nums):
    num_set = set(nums)
    longest = 0

    for n in num_set:           # iterate set to avoid processing duplicates
        if (n - 1) not in num_set:   # n is a sequence head
            length = 1
            while (n + length) in num_set:
                length += 1
            longest = max(longest, length)

    return longest
```

**Walkthrough for `nums = [100,4,200,1,3,2]`:**
- Set = {1,2,3,4,100,200}
- n=1: 0 not in set → head. Count: 1→2→3→4 → length=4
- n=2: 1 in set → skip
- n=3: 2 in set → skip
- n=4: 3 in set → skip
- n=100: 99 not in set → head. Count: just 100 → length=1
- n=200: 199 not in set → head. Count: just 200 → length=1
- Result: 4 ✓

**Complexity:** Time O(n) · Space O(n). Optimal.
